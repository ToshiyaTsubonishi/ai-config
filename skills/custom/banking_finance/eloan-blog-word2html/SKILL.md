# Word → HTML 変換スキル

## 目的
提供された Microsoft Word 原稿 (.docx) を、`assets/` ディレクトリ内の定型HTMLテンプレートに組み込む形で、Web サイト掲載用の正確な HTML に変換します。
Word の自由なフォーマット揺れに対応するため、Python でテキスト要素や画像を漏れなく抽出し、そのテキストベースの骨組みをもとに AI が最終的な HTML を組み立てます。

## 作業手順
1. **ブログの種類の確認（必須）**
   ユーザーから変換対象のファイルが提示されたら、まず **ブログのタイトルをユーザーに**確認します。ユーザーへの確認には ask user toolを使用します。
   ブログタイトルの選択肢は以下です：
   - ELタイムズ
   - bankloan
   - FPアドバイス

2. **変換対象とパラメータの特定**
   ユーザーから「Word 原稿 (.docx)」「画像パスのディレクトリ・連載番号（例: `80`, `115`等）」が提示されたことを確認します。不明な場合はユーザーに確認します。ユーザーへの確認には ask user toolを使用します

3. **ルールとテンプレートの読み込み**
   選択されたブログに応じたルールファイル（`references/`）とHTMLテンプレート（`assets/`）を読み込み、指示を理解します。
   - ELタイムズ: `references/el_times.md` & `assets/el_times_template.html`
   - bankloan: `references/bankloan.md` & `assets/bankloan_template.html`
   - FPアドバイス: `references/fp_advice.md` & `assets/fp_advice_template.html`

4. **コンテンツ抽出**
   `scripts/extract.py` を使用して Word ドキュメントからテキスト、画像、Adobe Stock URL、コメント（Alt属性）を抽出します。

5. **HTML アセンブル**
   抽出されたデータ、ルール、およびテンプレートに基づき、HTMLファイルを構成します。
   - テンプレート内の `<!-- CONTENT_START -->` と `<!-- CONTENT_END -->` の間に、抽出した内容をマークアップして挿入します。

6. **整合性確認**
   タグの整合性（閉じタグの過不足など）や禁止タグの有無を厳密にチェックし、最終的な HTML ファイルを出力します。

---

## 変換ルール（共通）

各ブログに共通する基本ルールです。詳細は各ブログ個別のルールファイルを確認してください。

- **文字コード**: UTF-8
- **改行コード**: LF
- **強調**: `<b>` はすべて `<strong>` に置換。絶対に省略しないでください。
- **全角スペース**: 段落先頭の全角スペース `　` は削除。

---

## 使用ツール
**この環境では python3ではなく、pythonコマンドを使ってください**
- **スクリプト**: `scripts/extract.py`
  - Word ドキュメントから構造化されたテキストデータを抽出します。
- **ルール定義**: `references/*.md`
  - 各ブログ記事特有のマークアップ、タグ、画像パスの定義です。
- **HTMLテンプレート**: `assets/*.html`
  - 各ブログ記事の共通ヘッダー・フッター・CSS定義を含むスケルトンです。
