# 証券・市場業務（アルゴ取引・マーケットメイク）運用マニュアル

## 1. HST（高速取引行為）オンボーディング・チェックリスト
- [ ] **HST認定**: 顧客の戦略が金商法第2条第44項の「高速取引行為」に該当するか否かの判定。
- [ ] **財務局登録**: 該当する場合、近畿財務局または関東財務局への登録済証を確認。
- [ ] **接続テスト**: サンドボックス環境での500ms以内のレイテンシ確認およびキルスイッチの作動試験。

## 2. トレーディング・リスク管理基準
- **ハードリミット**:
    - 単一注文数量上限 (Order Size Cap)。
    - 未約定注文の累計金額上限 (Total Notional Cap)。
- **キルスイッチ (Kill Switch)**:
    - 市場変動率（ボラティリティ）が閾値を超えた場合、またはシステム異常を検知した場合の全API注文の強制キャンセル。
- **過当投機・不公正取引監視**:
    - 見せ板（Layering）、スプーフィング、およびウォッシュトレードのAIリアルタイム検知。

## 3. マーケットメイク・インフラ
- [ ] **アローネット接続**: 低遅延通信（Co-location）およびFPGA等を用いたハードウェアアクセラレーションの運用。
- [ ] **流動性供給**: 対象銘柄におけるスプレッド維持および気配数量の提供義務（SLA）の遵守。

## 4. データ・エコシステム
- **Historical Tick Data**: 過去10年分以上の全板情報のAPI配信管理。
- **Developer Support**: Python/Go/Rust向け公式ライブラリの脆弱性管理。
