# Power Automate / PAD 人事自動化 障害復旧・保守手順書

## 1. 目的
人事業務の自動化フロー（Power Automate, Power Automate Desktop）において発生する「接続切れ」や「実行エラー」に対し、迅速な検知と復旧を行うための標準手順を定める。

## 2. 主要な障害原因と対策
| 障害事象 | 主な原因 | 恒久対策 |
| :--- | :--- | :--- |
| **接続が無効です** | 担当者のパスワード変更、MFA再認証 | サービスアカウント（共有メールボックス等）の使用、定期的な接続修復。 |
| **フローのタイムアウト** | 処理対象データ（Excel等）の肥大化、ネットワーク遅延 | データの分割処理（バッチ化）、Scopeアクションによるエラーハンドリング。 |
| **PAD 実行失敗** | UI要素の変更（システムの画面更新）、セッション切断 | セレクターの正規化、リトライポリシー（3回再試行等）の設定。 |

## 3. エラー通知・Try-Catch設計標準
全てのフローにおいて以下の「Try-Catch構造」を必須とする。
1.  **Scope: Try**: 本処理を実行。
2.  **Scope: Catch**: Tryが失敗した場合にのみ実行（`runAfter: failed`）。
    - **通知内容**: フロー名、エラー発生アクション名、実行履歴URL、エラーメッセージ（`result()`関数利用）。
    - **通知先**: Teamsの「人事DX保守チャネル」および担当者メール。

## 4. 復旧アクション (Step-by-Step)
1.  **実行履歴の確認**: 失敗した実行詳細を開き、エラーコードとメッセージを特定する。
2.  **接続の修復**: 「データ」>「接続」から赤い警告の出ている接続を選択し「接続の修復」を実行。
3.  **再送信**: 接続修復後、失敗した履歴から「再送信」をクリックし、データが二重登録されないか注意しつつ実行。
4.  **修正**: UI変更が原因の場合は、PADデザイナーでセレクターを再キャプチャする。

## 5. 監視とメンテナンス
- **週次チェック**: 成功率が95%を下回っているフローがないか、管理センターで確認。
- **28日ルール**: 削除されたフローは28日以内であればPowerShell（`Restore-AdminFlow`）で復元可能であることを認識しておく。
