# COMPANY HPM (就労管理) データ構造・連携仕様書

## 1. HPMモジュールの役割
COMPANY HPM (Human Performance Management) は、複雑な勤怠、工数、プロジェクト管理を一元化する。本エージェント（`hr-onboarding-sync-master`）は、入社時の初期設定をHPMへ自動反映させる役割を担う。

## 2. 主要データ・エンティティ (Schema)
| 項目名 | HPM内部コード | 設定値の例 / バリデーション |
| :--- | :--- | :--- |
| 勤務体系 | `WorkSchedule_ID` | `FLEX` (フレックス), `DISC` (裁量労働) |
| 承認ルート | `ApprovalPath_CD` | 部長 -> リーダー -> 本人 の階層コード |
| 休日カレンダー | `Calendar_GRP` | `HQ_CAL` (本社), `KANS_CAL` (関西) |
| 工数管理対象 | `Project_FLG` | `1` (管理する), `0` (管理しない) |

## 3. 入社時自動プロビジョニング手順
1.  **マスタ参照**: 採用時の職種・ランクに基づき、適切な `WorkSchedule_ID` をルックアップ。
2.  **API/CSV連携**: 生成された初期設定値を、HPMの「就労情報一括登録」インターフェースへ送信。
3.  **権限有効化**: CWS（Web Service）を通じたモバイル打刻の権限を `Status: Active` に更新。

## 4. 2026年 運用留意点
- **ハイブリッドワーク対応**: 在宅勤務比率に応じた「テレワーク手当」の自動計算フラグがHPM内で正しくセットされているかを確認。
- **長時間労働抑止**: HPMの閾値設定に基づき、入社1ヶ月目の残業時間が20時間を超えた際のアラート通知先を確認。
