# 入社・異動情報のデータフロー・ガバナンス定義

## 1. 概要
採用チームが獲得した内定者情報を、労務チームおよび情報システム部へシームレスかつ正確に橋渡しするためのデータ連携構造。

## 2. データ連携経路 (Data Flow)
1.  **採用システム**: 内定承諾情報をトリガーに、JSON/CSV形式でデータをエクスポート。
2.  **中間管理DB (SharePoint)**: `hr-onboarding-sync-master` がデータを正規化し、入社管理表へ統合。
3.  **COMPANY**: 確定した個人・雇用データをバッチ処理またはAPI（制限がある場合）で流し込み。
4.  **AD/IDP**: 入社日に基づき、アカウントの自動プロビジョニングを実行。

## 3. ガバナンス・セキュリティルール
- **個人情報の保護**: マイナンバーなどの高機密情報は、専用の収集ツール（COMPANYマイナンバー管理等）で直接収集し、中間のSharePointには保存しない。
- **データ整合性**: 氏名の旧字・新字の扱い、住所の表記揺れ（ハイフン等）は `Data Normalizer` エージェントがマスタ定義に基づき自動補正する。
- **権限管理**: 入社前の内定者情報へのアクセスは、採用チームと特定の労務担当者のみに限定する。

## 4. 2025年 法改正対応 (社会保険適用拡大)
- 週労働時間20時間以上のパート・アルバイト等の入社時、社会保険加入漏れがないよう、マスタ登録時に勤務条件による自動判定ロジックを組み込む。
