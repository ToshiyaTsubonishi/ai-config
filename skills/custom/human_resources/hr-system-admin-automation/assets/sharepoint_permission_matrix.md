# SharePoint 人事部 権限管理マトリクス (2025年版)

## 1. サイト・フォルダ権限設計
個人情報保護法に基づき、職責に応じたアクセス制限を定義。

| フォルダ名 | 管理対象データ | アクセス権限 (編集) | アクセス権限 (閲覧) | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| `01_採用情報` | 内定者履歴書、評価表 | 採用チーム、人事部長 | 役員、面接官(期間限定) | |
| `02_給与・振込` | 給与明細、振込データ、口座情報 | 給与担当、リーダー | なし | |
| `03_マイナンバー` | マイナンバー関連、身元確認書類 | 特定個人情報取扱者 | なし | COMPANY専用DB利用推奨 |
| `04_健康管理` | 健診結果、ストレスチェック、病歴 | 健康管理担当、産業医 | なし | 機微情報のため厳格管理 |
| `05_契約書原本` | 雇用契約書、誓約書 (締結済) | 手続きチーム、法務部 | 監査室、各担当者 | 電帳法対応フォルダ |

## 2. 権限付与・削除フロー (Power Automate連携)
1. **申請**: 依頼者が専用Formsで「期間」「目的」「対象フォルダ」を申請。
2. **承認**: 人事部長またはセキュリティ管理者がTeamsで承認。
3. **実行**: APIを介してSharePointの権限を自動付与。
4. **削除**: 指定期間終了後、または異動・退職時に自動的に権限を剥奪。
