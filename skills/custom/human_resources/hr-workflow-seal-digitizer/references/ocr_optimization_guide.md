# AI-OCR 精度向上・帳票設計ガイドライン

## 1. 概要
人事書類（履歴書、証明書、請求書等）をAI-OCRで読み取る際、エラーを最小化しデータ入力自動化を成功させるための設計指針。

## 2. 読み取りやすい帳票設計 (OCR Friendly)
1.  **枠線と余白**:
    - 文字記入欄には明確な枠（ボックス）を設け、隣接する文字との接触を防ぐ。
    - 読み取り対象項目の周囲には3mm以上の余白を確保する。
2.  **活字とチェックボックス**:
    - 自由記述を減らし、可能な限りチェックボックス [ ] や選択式にする。
3.  **フォントと色**:
    - 帳票タイトルや項目名は「ゴシック体」などの可読性の高いフォントを使用。
    - 記入欄の枠線はAIが除去しやすい「ドロップアウトカラー（薄い緑や青）」を推奨。

## 3. スキャンのベストプラクティス
- **解像度**: 推奨 200dpi〜300dpi。高すぎてもノイズを拾い、低すぎると文字が潰れる。
- **カラーモード**: グレースケールまたはカラーを推奨。白黒二値化は情報の欠落を招く場合がある。
- **前処理**: 傾き補正、ノイズ除去機能を有効にする。

## 4. エラー対策 (Exception Handling)
- **信頼度（Confidence Score）の活用**: AIが提示する信頼度が一定値（例：80%）を下回った場合、自動的に「要目視確認」フラグを立てるワークフローを構築する。
- **マスタ照合**: 読み取った氏名や社員番号を、即座にCOMPANYのマスタデータと突合し、不一致があればアラートを出す。
