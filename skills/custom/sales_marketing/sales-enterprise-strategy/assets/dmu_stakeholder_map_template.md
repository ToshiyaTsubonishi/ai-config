# DMUステークホルダーマップ＆分析テンプレート

| 項目 | 内容 |
| :---- | :---- |
| **氏名・役職** | 特定できた人物の正式な氏名と役職。 |
| **DMUにおける役割** | エコノミック・バイヤー、チャンピオン、ユーザー、インフルエンサー、ゲートキーパーのいずれか。 |
| **レポートライン** | 上司は誰か、部下は誰か。組織内での位置づけを把握する。 |
| **ビジネス仮説との関連性** | 我々が定義した主要なビジネス課題と、この人物の職務がどのように関連しているか。 |
| **想定される関心事（Pains/Gains）** | この人物が最も気にかけるであろうこと。（例：財務部門は総所有コスト（TCO）、IT部門はセキュリティ、ユーザーは操作性） |
| **影響力レベル（高・中・低）** | 意思決定に対する影響力の度合いを評価する。 |
| **関係性ステータス** | 良好、中立、否定的、未接触など、現在の関係性を記録する。 |
| **次のエンゲージメント・ステップ** | 具体的な次のアクションプラン。（例：「A氏からの紹介を依頼する」「コスト削減に関する事例を送付」「技術デモを調整する」） |
