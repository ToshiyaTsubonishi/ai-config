# Strategic Decision Matrix (SDM)

意思決定に迷った際、以下の基準に従って即断せよ。

## 1. Build vs Buy (作るか、買うか)

| 領域 | 判定基準 | 結論 |
| :--- | :--- | :--- |
| **Core Competence** (差別化の源泉) | 自社独自のデータやノウハウが必要か？ | **Build (内製)** |
| **Commodity** (誰でもできること) | 既に市場に優れたSaaSが存在するか？ | **Buy (調達)** |
| **Innovation** (未踏領域) | 技術が未成熟で、まだ誰も成功していないか？ | **Partner (提携/投資)** |

## 2. Human vs AI (誰がやるか)

| タスク特性 | 判定基準 | 担当 |
| :--- | :--- | :--- |
| **Creativity** (0→1) | 正解がなく、感性や倫理観が必要か？ | **Human (with AI)** |
| **Operations** (1→100) | 定型的で、スピードと正確性が求められるか？ | **AI Agent** |
| **Complex Logic** (Multi-step) | 複雑だが、論理的に説明可能か？ | **AI Chain (ReAct)** |

## 3. Go vs No-Go (やるか、やめるか)

| 評価軸 | 判定基準 |
| :--- | :--- |
| **Impact** | 10倍の改善が見込めるか？ (Incrementalな改善は却下) |
| **Speed** | 3ヶ月以内にMVPを出せるか？ (長期プロジェクトは分割) |
| **Risk** | 失敗した際、取り返しがつかないか？ (YesならGuardrail必須) |
