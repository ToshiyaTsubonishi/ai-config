# SBI AGI Design System Structure

AGI時代のデザインシステムは、静的なコンポーネントだけでなく、動的な振る舞いと倫理規定を包含する。

## 1. Foundation (Design Tokens)
*   **Core:** Color, Typography, Spacing, Iconography (SBI Brand aligned)
*   **AGI Specific:**
    *   **Trust Level:** 信頼度に応じたUIの「堅さ」の変動（例：雑談時は柔らかく、決済時は厳格に）。
    *   **Urgency:** 緊急度に応じたアラートレベルの定義。
    *   **Emotion:** AGIの音声・テキストのトーン＆マナー（Warm, Professional, Urgent）。

## 2. Dynamic Components
*   **Adaptive Cards:** コンテキスト（スマホ、ARグラス、車載ディスプレイ）に合わせて形状を変える情報カード。
*   **Voice Interfaces:** VUI（Voice User Interface）のためのスクリプトパターンとサウンドロゴ。
*   **Biometric Auth:** 顔、声、生体認証をシームレスに統合した認証フロー。

## 3. Experience Patterns
*   **Predictive Onboarding:** ユーザーの属性から初期設定を完了済みにするフロー。
*   **Conversational Transaction:** チャット/音声だけで完結する送金・投資フロー。
*   **Error & Recovery:** AGIの誤解釈をユーザーが自然に修正できるインタラクション。

## 4. Governance
*   **Ethical Guidelines:** ダークパターンの禁止、公平性の担保。
*   **Review Checklist:** AGIの挙動がデザイン原則に適合しているかの監査リスト。
