# インシデント対応プレイブック (Incident Response Playbook)

**対象:** CSIRTメンバー
**フェーズ:** 検知(Detect) -> 封じ込め(Contain) -> 根絶(Eradicate) -> 復旧(Recover)

## 1. 検知・分析 (Detection)
- [ ] **アラート確認:** Datadog/PagerDutyからの通知内容（エラー率急増、不正ログイン）を確認。
- [ ] **一次調査:** 影響範囲（顧客データ、サービス稼働）を特定し、Severity（Sev1-4）を判定する。
- [ ] **War Room:** Slack/Zoomで緊急対策本部を立ち上げる。

## 2. 封じ込め (Containment)
- [ ] **ネットワーク遮断:** 攻撃元のIPブロック、または感染端末の隔離。
- [ ] **アカウント停止:** 流出が疑われるID/Keyの無効化（Revoke）。
- [ ] **緊急停止 (Kill Switch):** 必要に応じてサービス全体または一部機能を停止する（スマートコントラクトのPause等）。

## 3. 根絶・復旧 (Eradication & Recovery)
- [ ] **原因特定:** ログ分析やフォレンジックにより侵入経路を特定し、パッチを適用する。
- [ ] **クリーンアップ:** バックドアやマルウェアが残っていないかスキャンする。
- [ ] **再開:** 正常性を確認した後、サービスを再開する。

## 4. 事後対応 (Post-Incident)
- [ ] **広報:** 影響を受けたユーザーおよび規制当局への報告（GDPR/APPI対応）。
- [ ] **再発防止:** ポストモーテムを作成し、恒久対策を実施する。
