# 目的

本スキルは、SaaS / ASP に関するヒアリング情報 （文章、箇条書き、仕様書メモ、既存の回答案など）から、**SaaS・ASPチェックシート**を作成することを目的とする。
本スキルで作成した **JSON** をもとに、`scripts/fill_xlsx.py` を実行することでSaaS・ASPチェックシートのテンプレート`assets/template.xlsx` に回答を転記し、作業は完了する。
**ask user tool**を積極的に利用しながら、SaaS / ASP に関するヒアリング情報をユーザーから取得すること。

# 入力
1.Webから収集できるSaaS / ASPの公式情報
2.ユーザーから与えられる SaaS / ASP に関する情報全般。**ask user tool**によりヒアリングする。
** 重要 : supplementに使用するため、SaaS導入の背景情報や補足事項を必ず ask user tool でユーザーに確認する**
例：
- セキュリティ仕様の説明文
- 箇条書きの運用ルール
- 仕様書や提案資料の抜粋
- 既存チェックシートの下書き

入力形式は問わない。
ユーザーから情報が与えられない場合、webから情報を収集する。

# 出力（必須）
ユーザーには、わかりやすいように表形式で出力内容を説明。

# ルール
- 対象は **No.1〜13のみ**（= Excelの16〜28行目）。それ以前の行は扱わない。
- `yes_no` は **必ず "YES" か "NO"** のどちらか（大文字）にする。
- `detail` は、チェック観点に対して **根拠や運用** が伝わる最小限の文章にする（長文にしない）。
- 入力情報から判断できない場合でも、`yes_no` を空にしない。  
  - 原則は **NO** に倒し、`detail` に「情報不足のためNO扱い」などの理由を書く。
- 生成AI関連（No.7〜9）は、入力に該当情報がない場合は NO に倒す（情報不足なら明記）。

# チェック項目（No → 観点）
1. 利用者の特定:サービスの利用者に一意のIDを割当て、個々の利用者を特定できるようにしていますか。
2. 利用者アカウント管理:不必要となった利用者アカウントは、速やかに失効させるため、人事部門等、関連部署から申請を受ける手順を定めていますか。（少なくともアカウントの棚卸は年１回実施すること）
3. 監視・追跡:インシデント時の調査のため、管理者・利用者のログの取得、または、サービス提供会社からのアクセス元IP、ID等の提供を得られますか。
4. なりすまし防止:なりすましを防止するため、ログインパスワードの他、２要素認証等によって、厳格に利用者を認証できますか。
5. 漏えい防止:許可なく任意に、利用者が他のSaaSにアプリ連携し、機密データを外部送信することを防止するため、連携を許可する設定は、管理者のみに限定していますか。※SaaSの機能拡張等により、変更が生じる場合があるため、少なくとも毎年度、再確認すること。
6. 契約終了後データの漏えい防止:サービス仕様、またはSLA(Service Level Agreement)等を確認し、契約終了時、サービス提供会社に預託した個人情報および機密情報が消去できますか。
7. 生成AIの利用①（生成AI機能の有無など）:サービスに入力または格納するデータを基に生成AIモデルが処理を行う機能は利用しますか。（Yesの場合、8および9を回答、Noの場合は8および9は回答不要）
8. 生成AIの利用②（利用する場合の制御・同意など）:生成AIに入力したデータは、サービス提供会社の他のユーザーが利用する生成AIモデルのトレーニングに利用されない仕様ですか。
9. 生成AIの利用③（学習利用・データ取り扱いなど）:生成AIへの入力データ、出力データに対してサービス提供会社の閲覧ができないよう制限されていますか
10. サービス提供者の信頼性の確認:サービス提供会社は、公的なセキュリティ認証を取得していますか。認証の例：プライバシーマーク／ISO-27001(ISMS)／ISO-27017(ISMSクラウドベンダ)／ISO-27018(ISMSクラウド個人情報保護)／NIST-SP800-53(FedRAMP)等
11. 認証情報の保護:SaaS環境上において保持する認証情報（パスワード）は、ハッシュ化または暗号化し、保護していますか。
12. セッション管理:ログイン認証後、サービス利用中の利用者を再確認するため、ログイン後24時間以内に再認証していますか。
13. データの所在地の確認:サービスで取扱うデータの所在地は日本国内のみですか。※SaaSのインフラ更新等により、変更が生じる場合があるため、少なくとも毎年度、再確認すること。

# Excel への転記方法
**重要: python3コマンドは許可されてない。pythonコマンドを使用すること**
本スキルで出力した JSON をファイル（例：input.json）として保存し、以下のコマンドを実行することで、テンプレート Excel に回答が転記されたファイルを生成できる。

python scripts/fill_xlsx.py \
  --input input.json \
  --output output.xlsx

- テンプレートは assets/template.xlsx を使用する。
- 転記対象は No.1〜13（16行目〜28行目）のみ。
- 既存の書式や結合セルは保持される
