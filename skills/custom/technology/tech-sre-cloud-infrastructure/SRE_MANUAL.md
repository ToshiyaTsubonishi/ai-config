# クラウド基盤・SRE & FinOps 運用マニュアル

## 1. FISC安全対策基準 準拠チェックリスト (Compliance)
- [ ] **Data Residency**: 顧客データの保存先が日本国内リージョンに限定されているか。
- [ ] **Encryption**: 保存時および転送時のデータ暗号化、ならびにBYOK（自社管理鍵）の運用。
- [ ] **Vulnerability Management**: クラウド基盤およびコンテナの脆弱性スキャン（CSPM/CWPP）の自動実行。
- [ ] **External Audit**: クラウドベンダーのSOC2レポート等の受領および評価。

## 2. FinOps コスト管理・最適化基準 (Cost)
- **Tagging Governance**: 全リソースへの所有者、環境、およびプロジェクトタグの付与強制。
- **Savings Optimization**:
    - RI（Reserved Instances）およびSavings Plansの適用率（ターゲット 80%以上）。
    - アイドルリソースの自動検知とAIによる削除/リサイズレコメンデーション。
- **Chargeback**: 部門別のクラウド利用料の正確な算出と月次レポート。

## 3. SRE & オブザーバビリティ・ワークフロー
1. **SLI/SLO Setting**: 可用性、レイテンシ、エラー率等のサービスレベル目標の策定。
2. **Error Budget**: SLO逸脱時のリリース停止および信頼性向上活動へのリソースシフト（アライメント）。
3. **Chaos Engineering**: 本番またはステージング環境における障害注入試験（ネットワーク断、プロセス停止等）の定例実施。

## 4. マルチクラウド & レジリエンス
- [ ] マルチクラウド（AWS+Azure等）間のデータ同期およびフェイルオーバー手順の整備。
- [ ] クラウドExit戦略（他クラウドへの移行性）の評価。
- [ ] オブザーバビリティ（AIOps）による障害予兆検知の精度検証。
